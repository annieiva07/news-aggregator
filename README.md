# Новостной агрегатор

Консольное Java-приложение для автоматического сбора, хранения, фильтрации и анализа новостей с RSS-лент различных источников.

## Возможности

- **Сбор новостей** — парсинг RSS-лент (ТАСС, Интерфакс, Вести.Ru), исключение дублей по URL
- **Автоматическое определение категории** — Политика, Экономика, Спорт, Наука, Культура
- **Автоизвлечение ключевых слов** — из заголовка и описания каждой статьи
- **Поиск и фильтрация** — по ключевым словам, категории, источнику, диапазону дат
- **Сортировка** — по дате, популярности (просмотры), источнику, категории
- **Аналитика** — топ ключевых слов за период, динамика упоминаний по дням
- **Статистика** — количество новостей по категориям
- **Экспорт** — в форматы CSV, JSON, HTML
- **Автообновление** — фоновый планировщик с настраиваемым интервалом
- **Управление источниками** — добавление и удаление RSS-источников во время работы
- **История изменений** — запись всех добавлений в таблицу `history`

## Запуск проекта

### Через IntelliJ IDEA

1. Открыть папку проекта в IntelliJ IDEA
2. Maven автоматически скачает зависимости
3. Запустить класс `com.newsaggregator.Main`

### Через JAR-файл

```bash
# Сборка
mvn package

# Запуск
java -jar target/news-aggregator-1.0-SNAPSHOT-jar-with-dependencies.jar
```

### Требования к окружению

- Java 17 или новее
- Доступ в интернет для парсинга RSS-лент

## Использование

После запуска откроется консольное меню:

```
===== НОВОСТНОЙ АГРЕГАТОР =====
1.  Показать все новости
2.  Поиск по ключевым словам
3.  Фильтр по категории
4.  Фильтр по источнику
5.  Фильтр по дате
6.  Сортировка новостей
7.  Статистика по категориям
8.  Аналитика и тренды
9.  Парсить новости
10. Автообновление [ВЫКЛ]
11. Управление источниками
12. Экспорт данных (CSV/JSON/HTML)
13. Добавить новость вручную
0.  Выход
```

### Первый запуск

1. Выбрать **пункт 9** — парсить новости
2. Выбрать источник или **«Все источники»**
3. Просматривать, фильтровать и анализировать собранные новости

### Автообновление

- Пункт **10** включает/выключает фоновый сбор новостей
- Интервал задаётся в минутах (по умолчанию 30)
- Работает в фоновом потоке-демоне, не блокируя интерфейс

### Экспорт данных

- Пункт **12** экспортирует все новости из базы
- Поддерживаемые форматы: **CSV**, **JSON**, **HTML**
- Файл сохраняется в текущей директории запуска

## База данных

Данные хранятся в файле `news_aggregator.db` (SQLite) в директории запуска.

### Таблица `news`

| Поле | Тип | Описание |
|------|-----|----------|
| id | INTEGER | Первичный ключ |
| title | TEXT | Заголовок |
| description | TEXT | Краткое описание |
| full_text | TEXT | Полный текст |
| publish_date | TIMESTAMP | Дата публикации |
| source_url | TEXT UNIQUE | URL статьи (уникальный — защита от дублей) |
| source_name | TEXT | Название источника |
| category | TEXT | Категория |
| keywords | TEXT | Ключевые слова (JSON) |
| media_urls | TEXT | Ссылки на медиафайлы (JSON) |
| views | INTEGER | Количество просмотров |
| created_at | TIMESTAMP | Дата добавления в БД |
| updated_at | TIMESTAMP | Дата последнего изменения |

### Таблица `history`

Хранит историю добавлений и изменений (аудит-лог).

## Архитектурные принципы

Проект построен по принципам **SOLID** и **GRASP**:

- **Single Responsibility** — каждый класс отвечает за одну задачу: `NewsRepository` — данные, `AnalyticsService` — аналитика, `SchedulerService` — таймер
- **Open/Closed** — новые парсеры добавляются через интерфейс `NewsParser` без изменения существующего кода
- **Liskov Substitution** — `RssParser`, `RiaParser`, `YandexNewsParser` взаимозаменяемы через `NewsParser`
- **Dependency Inversion** — `ParserManager` зависит от абстракции `NewsParser`, а не от конкретных классов
- **Information Expert (GRASP)** — `NewsRepository` сам знает, как работать с базой данных
